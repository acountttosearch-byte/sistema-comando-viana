1. VISÃO GERAL DA APLICAÇÃO
================================================================================

Nome: Sistema Comando Viana
Tipo: Aplicação Web para Gestão de Ocorrências Policiais
Tecnologia Principal: Laravel 12.x Framework
Padrão Arquitetônico: MVC (Model-View-Controller)
Tipo de Interface: REST API + Web Interface
Autenticação: Laravel Sanctum
Build Tool: Vite v7.0.7
CSS Framework: Tailwind CSS v4.0.0

Funcionalidades Principais:
• Gestão de Ocorrências (registos, estados, investigação)
• Gestão de Detenções (registos, estados, liberação)
• Gestão de Investigações (investigadores, notas, progresso)
• Gestão de Alertas (BOLO - Be On the LookOut)
• Gestão de Evidências com Cadeia de Custódia
• Gestão de Patrulhas e Incidentes
• Gestão de Recursos (Viaturas, Armamento)
• Sistema de Mensagens Internas
• Relatórios Operacionais
• Dashboard com Métricas
• Sistema de Permissões baseado em Perfis

FLUXO DE LOGIN:
────────────────────────────────────────────────────────────────────────────
1. Usuário acessa GET  / ou GET  /login
   → Redireciona para página de login (view: auth/login)

2. Usuário submete POST /login com email e password
   → Validação de emails e passwords obrigatórios
   → Tentativa de autenticação com Auth::attempt()
   → Se bem-sucedido:
      • Regenera session token
      • Atualiza último_acesso e IP
      • Registra login em Log
      • Redireciona para /painel
   → Se falhar: retorna com erro "Credenciais inválidas"

3. Acesso protegido a /painel e API endpoints
   → Middleware 'auth:web' valida sessão
   → Acesso negado sem autenticação


PROTEÇÕES:

✓ Passwords hasheadas (Laravel default)
✓ Regeneração de token CSRF após login
✓ Registro de último acesso e IP
✓ Logout com invalidação de sessão
✓ Proteção em rotas autenticadas
✓ Dois níveis de permissão: Perfil e Permissão detalhada

18. POSSÍVEIS OCORRÊNCIAS DE ERRO
================================================================================

ERRO 401 - NÃO AUTENTICADO:
────────────────────────────────────────────────────────────────────────────
Quando: Acessar endpoint sem estar logado
Solução: Fazer login primeiro
Endpoint exemplo: GET /api/dashboard/metricas

ERRO 403 - PERMISSÃO NEGADA:
────────────────────────────────────────────────────────────────────────────
Quando: Usuário logado mas sem permissão
Solução: Verificar perfil do usuário
Endpoint exemplo: GET /api/logs (requer permissão 'logs.ver')

ERRO 404 - NÃO ENCONTRADO:
────────────────────────────────────────────────────────────────────────────
Quando: Recurso não existe
Solução: Verificar ID e rota
Exemplo: GET /api/ocorrencias/9999

ERRO 422 - VALIDAÇÃO:
────────────────────────────────────────────────────────────────────────────
Quando: Dados inválidos enviados
Solução: Verificar erros retornados
Exemplo: POST /api/ocorrencias com tipo_crime_id inválido

ERRO 500 - ERRO INTERNO:
────────────────────────────────────────────────────────────────────────────
Quando: Erro no servidor
Solução: Verificar logs (artisan tail ou storage/logs)
Debug: APP_DEBUG=true (apenas desenvolvimento)

ERRO COMUM - CSRF TOKEN:
────────────────────────────────────────────────────────────────────────────
Quando: POST sem token CSRF (web forms)
Solução: Incluir token no form: @csrf

ERRO COMUM - SOFT DELETES:
────────────────────────────────────────────────────────────────────────────
Quando: Dado deletado logicamente não aparece
Solução: Usar withTrashed() ou onlyTrashed() em queries

ERRO COMUM - N+1 QUERIES:
────────────────────────────────────────────────────────────────────────────
Quando: Muitas queries executadas
Solução: Usar eager loading com with()
Exemplo: Ocorrencia::with(['tipoCrime', 'estado', 'unidade'])->get()

ERRO COMUM - PERMISSÃO DE ARQUIVO:
────────────────────────────────────────────────────────────────────────────
Quando: Erro ao criar arquivo de cache ou log
Solução: Verificar permissões de pasta storage/ e bootstrap/cache/
chmod -R 775 storage bootstrap/cache

ERRO COMUM - BANCO DE DADOS:
────────────────────────────────────────────────────────────────────────────
Quando: Conexão recusada
Solução: Verificar credenciais .env
• DB_HOST
• DB_PORT
• DB_DATABASE
• DB_USERNAME
• DB_PASSWORD

ERRO COMUM - MIGRATIONS NÃO RODADAS:
────────────────────────────────────────────────────────────────────────────
Quando: Tabelas não existem
Solução: php artisan migrate --force
